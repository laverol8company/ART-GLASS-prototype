## 1. Проект и цель

Art Glass — студія ремонту й полірування **автоскла** (ремонт сколів і тріщин, полірування скла від подряпин, полірування фар, захисна плівка). Бренд монохромный: чёрный круг, тонкая серебряная line-art машина (клиновидный суперкар), serif-вордмарк ARTGLASS.

- **Реальная аудитория сайта** — владельцы авто, премиум-сегмент Киева.
- **Реальный «зритель» прототипа** — владелец студии, которого надо вау-нуть.
- **Питч-логика страницы:** біль (скол/тріщина на склі, мутні жовті фари, подряпини на склі) → доказ (до/після) → що робимо (послуги) → як (процес) → довіра → запис за 30 секунд через бота.

## 4. Дизайн-система (design tokens)

### 4.1. Палитра — монохром + дисциплінований функціональний акцент

Кадром правят графит/хром/кость; головний акцент — свет/стекло (specular, sheen, хром). Дозволено лише **два функціональні кольори в межах ≤5%**: тепле **champagne-світло** (як світло фар) і **зелений лише як статус «відчинено»**. Червоний, неон і будь-яка пістрявість — заборонені (антипатерн ніші).

```css
@theme {
  --color-bg:        #0B0C0E; /* фон */
  --color-bg-2:      #0C0D0F;
  --color-surface:   #131417; /* поверхности */
  --color-surface-2: #16181C;
  --color-chrome:    #C6CACF; /* серебро/металл (линии, кромки) */
  --color-chrome-2:  #9A9EA6;
  --color-bone:      #ECECEE; /* основной текст, светлые CTA */
  --color-muted:     #9A9EA6; /* вторичный текст */
  --color-muted-2:   #6E727A; /* подписи, лейблы */
  --color-line:      #1E2024; /* разделители */
  --color-line-2:    #3A3D42;

  --color-accent:    #E7C79A; /* тепле світло фар — лише блики/цифри, ≤5% */
  --color-status:    #6FCF97; /* функціональний «відчинено» */

  --font-display: "Clash Display", "Neue Montreal", sans-serif;
  --font-body:    "Geist", "Hanken Grotesk", sans-serif;
  --font-mono:    "Geist Mono", "JetBrains Mono", monospace;

  --radius-sm: 10px;
  --radius-md: 14px;
  --radius-lg: 22px;
  --radius-pill: 999px;
}
```

### 4.2. Типографика

- **Display:** Clash Display (или Neue Montreal). **Body:** Geist (или Hanken Grotesk). **Mono:** Geist Mono / JetBrains Mono.
- **НЕ использовать:** Inter (как дефолт), Space Grotesk (бренд Laverol + переизбыток в AI-выдаче), Instrument Serif / Fraunces. Serif в вёрстке запрещён — serif живёт **только в самом логотипе** Art Glass.
- Трекинг дисплея плотный (`-0.02em` и плотнее на крупных кеглях).
- **Цифры — табличные** (`font-variant-numeric: tabular-nums`) на счётчиках, ценах, спеках.
- Ширина строки **60–70 знаков** (`max-width: 60ch–70ch`).
- Украинские кавычки «», тире в диапазонах — `–` (en-dash), не em-dash как приём.
- Неразрывные пробелы: `1 240 ₴`, перед короткими предлогами/союзами, чтобы не было висячих и сирот.

### 4.3. Spacing & layout

- Жёсткая **8pt-сетка**. Контейнер max ~1280px. Щедрый, осмысленный воздух.
- Асимметричные редакторские развороты вместо «всё по центру».

### 4.4. Тени и elevation

Тонированы под фон, не чистый чёрный:

```css
--shadow-1: 0 1px 0 rgba(255,255,255,.06) inset;
--shadow-2: 0 8px 32px rgba(0,0,0,.35);
```

### 4.5. Язык движения (motion)

```css
--ease-out:   cubic-bezier(.22,1,.36,1);  /* входы секций */
--ease-inout: cubic-bezier(.76,0,.24,1);  /* стеклянная завеса */
--ease-glide: cubic-bezier(.4,0,.2,1);    /* блики, sweep */
```

- Emphasis-доводка через spring (motion): `{ type:"spring", stiffness:120, damping:18 }`.
- Длительности 0.4–0.9 с. **Не анимировать `blur`** (только transform/opacity).
- **Везде fallback на `prefers-reduced-motion`** — переходы схлопываются в статику.

## 5. Liquid glass — материал-акцент

Это центральный материал бренда (Art Glass). Эстетика iOS 26: полупрозрачное стекло, преломляющее фон, тонкая светящаяся кромка, блик.

### 5.1. Базовый рецепт (tier 1, чистый CSS)

```css
.liquid-glass{
  position:relative; overflow:hidden;
  background:rgba(255,255,255,.04);
  backdrop-filter:blur(16px) saturate(150%);
  -webkit-backdrop-filter:blur(16px) saturate(150%);
  box-shadow:0 8px 32px rgba(0,0,0,.4),
             inset 0 1px 1px rgba(255,255,255,.22),
             inset 0 -1px 1px rgba(255,255,255,.05);
  border-radius:var(--radius-lg);
}
/* тонкая светящаяся кромка (mask-composite) */
.liquid-glass::before{
  content:""; position:absolute; inset:0; border-radius:inherit; padding:1.3px;
  background:linear-gradient(135deg, rgba(255,255,255,.6),
            rgba(255,255,255,.08) 38%, rgba(255,255,255,0) 55%, rgba(255,255,255,.45));
  -webkit-mask:linear-gradient(#fff 0 0) content-box, linear-gradient(#fff 0 0);
  -webkit-mask-composite:xor; mask-composite:exclude; pointer-events:none;
}
/* specular за курсором: child-слой radial-gradient at var(--mx) var(--my) (JS пишет --mx/--my на mousemove) */
/* sheen: скользящий блик, transform translateX (только transform/opacity) */

/* твёрдый фростед-фоллбэк */
@media (prefers-reduced-transparency:reduce){
  .liquid-glass{ background:#1A1C20; backdrop-filter:none; -webkit-backdrop-filter:none; }
}
```

### 5.2. Три тира точности (по месту)

1. **CSS** (рецепт выше) — быстро, лёгко, база. Ставим везде, где нужно стекло.
2. **SVG-преломление** (`feDisplacementMap` + `feTurbulence`) — настоящее искажение фона по краям. Тяжелее, точечно на 1–2 акцентах.
3. **WebGL/шейдер** — максимум физики, **только один** hero-момент.

### 5.3. Где применять (точечно, НЕ везде)

Floating nav, карточка и кнопка бота, главный CTA, мобильное меню-оверлей, ручка слайдера до/после, hero-бейдж. **Не заливать стеклом каждую секцию** — перформанс и читаемость.

### 5.4. Правила стекла

- Под текстом на стекле — лёгкий скрим, чтобы контраст не падал ниже **AA**.
- Лимит стеклянных слоёв в одном кадре (перформанс).
- На мобилке и при `prefers-reduced-transparency` — твёрдый фростед.
- Не анимировать `backdrop-filter`/blur.

## 6. Архитектура страницы (12 блоков + global)

Чередуем layout-семейства; не больше 2 «зигзагов» подряд; eyebrow-лейблы экономно.

### Global (сквозное)

- **Счётчик секций** `01 / 12` сбоку, трекает скролл (как Radian).
- **Плавающий блок действий:** Telegram + «Записатися», появляется после hero, всегда под рукой на мобилке.
- **Шапка-трансформер:** прозрачная над hero → фростед-стекло на скролле; sticky.
- **Мобильное меню:** полноэкранный стеклянный оверлей со списком секций и счётчиком.
- **Back-to-top**, плавный якорный скролл (Lenis).

### 6.1. Hero — full-bleed cinematic

- Фон: видео процесса (poster-кадр для LCP, видео — lazy) либо тёмный кадр авто.
- Линия авто из лого «рисует себя» (animated SVG line-draw) + specular-блик проезжает по кузову.
- Заголовок (kinetic reveal, по словам). Подзаголовок ≤20 слов. **1 главная CTA** «Записатися» (магнитная) + Telegram. Scroll-cue.
- Копірайт (реалізовано в прототипі hero): H1 — «Чистий погляд на дорогу.» Підзаголовок — «Ремонт сколів і тріщин, полірування скла та фар, захисна плівка. Ловимо скол, поки він не став тріщиною — у Києві.»

### 6.2. Stats band — полоса доверия

- Горизонтальная лента, count-up на въезде, табличные цифры: «1 240+ авто · 7 років · 36 міс гарантії». Числа живые (не 99.99%).
- Layout: одна строка с тонкими разделителями, НЕ три карточки.

### 6.3. Проблема — editorial

- Один сильный макро-кадр пошкодженого скла/фари (скол, тріщина, каламуть) + плотный текст. Reveal-маска, текст стаггером.
- Копірайт-напрям: маленький скол швидко стає тріщиною; мутні фари = слабке світло й небезпека вночі; подряпини на склі б'ють по огляду.

### 6.4. До / Після — интерактив (гвоздь)

- Перетягиваемый слайдер-сравнение, clip-path на `useMotionValue` (НЕ useState на кадр). Ручка — стеклянная (liquid-glass).
- Несколько пар (скол → відремонтований; мутна жовта фара → прозора; подряпина → відполіроване скло). Переключатель или scroll-snap. Опц. scroll-pinned «протирання».

### 6.5. Послуги — sticky scroll-synced list + counter (приём Radian)

- Залипающий список услуг, медиа меняется синхронно, счётчик `01/04`. Альтернатива — accordion-strips (НЕ 3 равные карточки).
- Услуги (UA, реальні Art Glass): 
  - Ремонт сколів і тріщин — «Зупиняємо тріщину. Відновлюємо міцність і прозорість скла — без заміни.»
  - Полірування автоскла — «Прибираємо подряпини та помутніння. Чистий огляд без бліків.»
  - Полірування фар — «Знімаємо жовтизну й каламуть. Яскравіше світло — безпечніша їзда.»
  - Захисна плівка — «Бронюємо фари та зони ризику від сколів і подряпин.»

### 6.6. Процес — горизонтальный scroll-pan (pinned)

- Этапы едут вбок, пока скроллишь: **Огляд і замір → Підготовка зони → Ремонт / Полірування → Полімеризація → Контроль під світлом**.
- Без ярлыков «Етап 1 / Step 1» — label = само действие.

### 6.7. Наші роботи — masonry

- Сетка разной высоты, hover-zoom, подпись (модель/услуга) появляется снизу, НЕ плашкой поверх фото. Опц. drag-to-pan.

### 6.8. Про студію / Чому ми — asymmetric / divide-y

- **Майстер/команда з іменами** (или владелец) + опорное фото студии/процесса — один из сильнейших триггеров доверия в нише. Премиум = студия/оборудование; SMB = история мастера. Короткая история «почему нам можно доверять».
- **Гарантія + бейджи** під автоскло (під реальні дані Art Glass — плейсхолдер): гарантія на ремонт скола (тріщина не піде далі), плівка — бренд (напр. XPEL / SunTek, до 10 років), полірування фар/скла з контролем результату. **Гарантійні умови — прямо на картки** послуг у §6.5.
- Контроль під лампою, прозорість матеріалів і процесу. Крупная типографика + одно опорное фото. Без трёх равных карточек.

### 6.9. Відгуки — carousel/marquee

- Встроенные **Google-відгуки з рейтингом + кількістю** (напр. 4.9 ★, 120+ відгуків — реалістично, **НЕ ідеальні 5.0**). UA-имена (вымышленные). **Имя майстра в отзывах** усиливает доверие («дякуємо майстру …», «питайте …»).

### 6.10. FAQ — accordion

- UA-питання: чи можна відремонтувати скол (і коли вже треба міняти скло), чи піде тріщина далі після ремонту, скільки тримається полірування фар, догляд після, строки робіт, гарантія.

### 6.11. Локація + контакти — split

- Встроенная **Google Maps в тёмной кастомной стилизации**. Адрес/години/«зараз відчинено». Tap-to-call, tap-to-route.

### 6.12. Бот-кваліфікатор + запис

- Стеклянная карточка + плавающий стеклянный лаунчер. См. §8.

### Footer

- Контакти, соцмережі, **Telegram**, навигация. OG/мета (см. §10).

## 7. Анимации и скролл-приёмы (библиотека)

### Входы блоков (entrances)

- **Скляна завіса** — фростед-панель уходит вверх, открывая секцию. Фирменный переход между ключевыми блоками.
- **Світловий змах** — specular-блик проезжает и «проявляет» контент (mask reveal).
- **Підйом + фокус** — контент всплывает из blur в резкость с пружинным доводом. Дефолт большинства секций.
- **Каскад** — дочерние элементы поднимаются по очереди (stagger) + count-up.

### Скролл-хореография

- **Sticky-stack** — секции пинятся, предыдущая уезжает вглубь с затемнением (услуги).
- **Горизонтальный проезд** — процесс.
- **Параллакс-глубина** — слои «авто / свет и блики / графит» на разной скорости (hero + редакторские секции).
- **Счётчик секций** `01/12`.
- **Покадровый скраб** (опц., требует ассета) — 360° оборот готового авто или «брудна → чиста».

Один easing-язык на всё. `prefers-reduced-motion` → всё в статику.

## 8. Бот-квалификатор

### Слой 1 — детерминированный (мгновенный, без затрат)

Шаги: **тип авто → проблема → бажаний результат → строки.**

| Проблема (вход) | Результат (вход) | Рекомендація (выход) |
|---|---|---|
| скол / тріщина на лобовому | зберегти скло, без заміни | Ремонт скола/тріщини (поки не пішла далі) |
| подряпини / помутніння скла | чистий огляд | Полірування автоскла |
| мутні / жовті фари | яскравіше світло | Полірування фар (+ опц. захисна плівка) |
| захист фар / зон ризику | захист від сколів | Захисна плівка |

На выходе: рекомендованная услуга + price band (₴, плейсхолдер) + предлагаемые слоты.

### Слой 2 — AI (опционально, вау-демо)

Поле «опишіть своїми словами» → вызов Anthropic API (Claude) интерпретирует и рекомендует услугу с человеческим обоснованием.
- **Прод:** ключ только через серверный прокси (route handler), не на клиенте.
- **Прототип:** можно демо-вызов для презентации.

### Запись = хендофф в Telegram

Финальный шаг открывает Telegram с префилл-сводкой:

```
https://t.me/ARTGLASS_PLACEHOLDER?text=<urlencoded: «Авто: …; Послуга: …; Бажаний час: …»>
```

В проде → реальный календарь / мини-CRM Laverol.

### UI бота

Стеклянная карточка (tier 1–2), плавающий лаунчер, step-переходы (motion), состояния: loading / success / error («не зрозумів, уточни»), confirmation-toast.

## 10. Премиум-крафт (expensive signals)

- **Type & rhythm:** см. §4 — табличные цифры, measure 60–70ch, UA-кавычки, неразрывные пробелы, без сирот, 8pt-сетка.
- **Единый грейд фото (огромный рычаг):** все кадры к одному графику — десатурация + холодный нейтральный каст + лёгкое зерно + единый контраст; опц. дуотон graphite/chrome. Макро-детали: капли на лаке, отражение лампы. Разнородные фото из инсты начинают выглядеть как один дорогой бренд.
- **Авторская подпись:** короткий intro-loader (линия авто рисует себя + проявляется имя, **один раз**); линия из лого как сквозной мотив (разделители, водяной знак, элемент hero); асимметричные развороты.
- **Доверие:** люди/мастер с именами; прозрачность процесса/материалов/сертификаты/гарантия; гайд по уходу после; блок «Авто, які ми вкривали» (лого премиум-марок, placeholder).
- **Профессионализм:** корректные OG/Twitter cards + favicon + web manifest (чтобы ссылка при шере выглядела как продукт); спроектированные 404 и thank-you; один язык easing; скорость как часть премиума.

## 12. Перформанс — обязательно

## 16. Жёсткие запреты (anti-slop) — НЕ ДЕЛАТЬ

- ❌ Кастомный курсор. Неон / outer glow. AI-фиолет. **Червоний** та «гоночні» градієнти. Пістрявість / багатоколірність / мигання. Дозволений виняток: тепле champagne-світло ≤5% + зелений лише як статус «відчинено»; усе інше — поза палітрою лого.
- ❌ Три одинаковые карточки в ряд. Ярлыки «Етап 1 / Step 1».
- ❌ Inter (как дефолт), Space Grotesk, Instrument Serif / Fraunces; serif в вёрстке.
- ❌ Em-dash как приём; филлер-копирайт («можливості без меж», «без зобов'язань» и т.п.); generic-нейминг «Studio».
- ❌ Заимствованные внешние видео-стримы (чужой Mux и пр.).
- ❌ Фейково-идеальные числа (99.99%); плашки-пилюли поверх фото для подписей.
- ❌ Анимировать blur; заливать стеклом всё подряд; жертвовать читаемостью или скоростью ради вау.
